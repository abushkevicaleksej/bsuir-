Оформление.

В файле можно использовать комментарии, начинающиеся с “//” либо заключённые в “/*” “*/”. Встречающиеся в комментариях зарезервированные слова m4scp необходимо заключать в квадратные скобки. Файл должен начинаться с комментария-заголовка следующего вида:

 

/////////////////////////////////////////////////////
//        Файл: <имя файла>
//    Описание: <краткое описание содержимого файла>
/////////////////////////////////////////////////////
//       Автор: <ФИО автора>
//        Дата: <дата>
//              <внесённые изменения>

После краткого описания рекомендуется указывать сегмент, в который планируется размещать содержимое файла. Указание на размещаемый сегмент должно быть оформлено в виде следующей строки.

//  Размещение: <идентификатор>    

При добавлении/внесении изменений, вносящий указывает свои ФИО, если они не совпадают с ранее указанными, пишет дату и описывает внесённые изменения:

// Разработчик: <ФИО разработчика>
//        Дата: <дата>
//              <внесённые изменения>

Разработчик должен указывать свои ФИО полностью.

Все комментарии к алгоритму за исключением комментариев к константам, переменным и параметрам должны начинаться с начала строки и не превышать 65-й позиции.

Должны быть прокомментированы все переменные, константы и формальные параметры.

Рекомендуется писать комментарии в алгоритме уже через каждые 3-4 оператора. Если это необходимо, то можно писать комментарии на каждый оператор. Если файл содержит несколько процедур, то перед каждой процедурой необходимо детализировано описать её назначение, которое не должно дублировать краткое описание содержимого файла.

Файлы, содержащие только описание пар синонимов, должны иметь расширение scsy. Во включениях можно указывать только scs либо scsy файлы.

Пример оформления программы в формате m4scp:

/////////////////////////////////////////////////////
//        Файл: test.m4scp
//    Описание: файл содержит программу для
//              тестирования подпрограммы выявления
//              тождественных понятий
/////////////////////////////////////////////////////
//       Автор: IVP
//        Дата: 1.05.2002

#include "scp_keynodes.scsy"

#include "compare_conceptions.scs"

 

/////////////////////////////////////////////////////
// текст программы тестирования

program(test,
        [[ a = {c,d,e,f,g},    // первое понятие
           b = {c,d,e,f,g},    // второе понятие
           compare_conceptions // тестируемая подпрограмма
        ]],
        [{ seta,               // множество отличий a от b
           setb,               // множество отличий b от a
           descr               // дескриптор процесса
        }])

 

/////////////////////////////////////////////////////
// сравниваем a и b

callReturn([1_: fixed_: compare_conceptions,
            2_: fixed_: [{
                          1_: a,
                          2_: b,
                          3_: seta,
                          4_: setb
                        }]],descr)

 

/////////////////////////////////////////////////////
// если есть отличия в a, то выводим их

ifVarAssign([1_: seta],,test_setb)

printEl([1_: fixed_: seta])

 

/////////////////////////////////////////////////////
// если есть отличия в b, то выводим их

label(test_setb)

ifVarAssign([1_: setb],,test_exit)

printEl([1_: fixed_: setb])

 

label(test_exit)

return()

end
